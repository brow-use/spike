App management

Register and switch between web applications before recording

Overview

All recording skills — record-page-objects and record-workflow — start by reading the currently active app to know which URL to work with and what the app does. The app management skills are how you register applications and set which one is active.

Apps are persisted locally. You register an app once and switch between them as needed — you do not need to re-register every session.

Skills

/bu:create-app
Register a new application. Asks for a name, URL, and description, then saves it.
The description is used by recording skills to understand the app's purpose — write something meaningful, not just the app name.
/bu:set-current-app
Switch to a registered app and navigate the browser to its URL.
Run this before any recording skill. It sets the active app context and opens the browser at the right starting point.
/bu:list-app
Show all registered apps as a table with id, name, URL, and description.
/bu:delete-app
Remove a registered app. Shows the app list, asks which to delete, and asks for confirmation before removing.

Typical setup sequence

For a new application:

/bu:create-app       ← register the app once
/bu:set-current-app  ← activate it and open the browser
/bu:record-page-objects

In subsequent sessions, only /bu:set-current-app is needed — the app is already registered.
