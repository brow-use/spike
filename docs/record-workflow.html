record-workflow

Record a reusable workflow function by performing steps live in the browser

Overview

This skill records a user-described sequence of browser interactions and writes it as a reusable TypeScript async function. Unlike record-page-objects, which captures the structure of pages, this skill captures a flow — a named sequence of actions that can be called from any test.

The agent loop, MCP server, and browser automation mechanism are the same as described in record-page-objects. What differs is the purpose of the run and what gets written at the end.

What makes it different

record-page-objects maps the structure of an app — what pages exist and what elements are on them. record-workflow maps a behaviour — a specific sequence of steps a user performs to accomplish something. The output is not a class but a function: self-contained, parameterised for variable inputs (credentials, search terms, item names), and importable directly into a test.

Workflows are also the composition layer between page objects and tests. The generated function imports and uses page object classes from ../page/, so page objects define the vocabulary and workflows define the sentences.

Tracing

The skill wraps the entire browser session in a Playwright trace. Before the first step, start_trace begins recording — capturing screenshots, DOM snapshots, and network activity at every interaction. When the scenario is complete, stop_trace saves the trace as a .zip to output/trace/.

The trace is a debugging artefact. It lets you replay exactly what the agent did step by step, inspect the DOM at any point, and verify that the recorded workflow reflects what actually happened in the browser.

Tools used

perception get_accessibility_tree snapshot action navigate click type trace start_trace stop_trace output write_workflow

snapshot is used here in addition to get_accessibility_tree. Where the accessibility tree gives structure for choosing selectors, snapshot captures a visual state after significant changes — useful for verifying that an action had the expected effect before proceeding.
